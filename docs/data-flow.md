# Data Flow

## End-to-end flow

```text
External Sources
  ├─ Official (NSE/IR/regulator)
  ├─ RSS/news/forum
  ├─ HTML listing/pages
  ├─ Sitemap discovery
  └─ APIs (fx/social/etc.)
        |
        v
A/B/C ingestion pipelines
  ├─ normalization + dedupe
  ├─ scoring/classification/theme inference
  └─ source health + run metrics
        |
        v
Service-scoped Postgres tables
  ├─ A outputs (prices/briefings)
  ├─ B outputs (announcements)
  ├─ C outputs (sentiment)
        |
        v
D synthesis (A+B+C)
        |
        v
E pattern memory (from D + context)
        |
        v
F narratives (from A–E + evidence jobs)
        |
        v
Gateway APIs + WS monitor feeds
        |
        v
Next.js dashboard + email artifacts
```

## Run lifecycle flow

```text
Scheduler/Gateway manual trigger
  -> publish RunCommandV1 (Redis)
  -> agent service consumes command
  -> start_run() emits RunEventV1(running)
  -> pipeline executes
  -> finish_run()/fail_run() emits terminal RunEventV1
  -> RunLedger consumes and persists timeline
```

## Lane flow (Kenya vs Global)

```text
source.scope
  kenya_core      -> primary feed behavior
  kenya_extended  -> contextual Kenya lane
  global_outside  -> impact-scored context lane

global_outside + kenya_impact_score >= threshold
  -> promoted_to_core_feed=true
else
  -> visible in global lane only
```

## Email flow (A/B/C first)

- Daily digest: generated by Agent A from A+B+C+F context.
- High-impact alerts: generated by Agent B based on alert gating.
- Weekly sentiment legacy mail: Agent C (depending on digest cutover flags).
- Validation runs: scheduled ops task writes to RunLedger validation tables.
